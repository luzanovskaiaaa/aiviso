# Aiviso — правила для Claude

Этот файл — обязательные правила работы над проектом. Алёна — селлер, не разработчик. Цель — продукт уровня Pixelcut / Photoroom для российского рынка WB/Ozon.

## ОБЯЗАТЕЛЬНО при ЛЮБЫХ правках UI/дизайна

**Перед** изменением любого визуального элемента (страницы, компоненты, секции, мокапы, иконки, типографика, цвета, отступы) — **ВНАЧАЛЕ** применяй skill `ux-designer` (он лежит в `.claude/skills/ux-designer/SKILL.md` этого проекта).

Это правило без исключений. Даже если правка кажется тривиальной («просто добавить кнопку», «просто заменить текст»), сначала сверяйся со skill'ом, потом правь код.

Если skill уже применялся в этой сессии — повторно загружать не нужно, но ход решения должен опираться на его принципы.

## Запрещено в продакшене

### Эмодзи как иконки
**НЕЛЬЗЯ** использовать эмодзи (`📸 💬 ⚡ 🔗 ✨ 📦 🎁` и любые другие) в качестве иконок UI на лендинге, в кабинете, в письмах. Эмодзи выглядят как «наскальная живопись», ломают восприятие профессионального продукта, по-разному рендерятся на разных ОС.

**ВМЕСТО** — line-art SVG иконки в фирменном стиле:
- цвет stroke: `#7c3aed` (бренд) или `#9ca3af` (серый neutral)
- `strokeWidth: 1.6-2`, `strokeLinecap: round`, `strokeLinejoin: round`
- Размер 16-24px внутри кружка-фона `border-radius: 10-12px`, фон `#f5f3ff` (бренд) или `#f3f4f6` (нейтральный)

В качестве источника готовых иконок — Lucide (MIT, уже подключён через `api/app/services/icons/`). Можно копировать SVG-пути из <https://lucide.dev>.

### Эмодзи в мокапах / скриншотах
В стилизованных интерфейсных мокапах (как мокап-чат на лендинге) **нельзя** ставить эмодзи в качестве «фото товара» или «результата генерации». Это смотрится как заглушка ради заглушки.

**ВМЕСТО** — стилизованный плейсхолдер:
- мини-плашка с надписью «фото» в углу
- line-art силуэт продукта (3:4 коробка / силуэт одежды / etc) на нейтральном градиенте
- цветовая дифференциация «до/после»: серый градиент для исходника, фирменный фиолетовый для результата

В идеале — реальные фото-примеры (как в `/media/_landing/before_after/`). Эмодзи — только в крайнем случае черновика.

### Символы которыми не злоупотреблять
- `→` (стрелка) — допустимо в CTA-кнопках, но не более 1 раза на блок
- Пунктуация: `·` (middle dot) для разделения метаданных — ок; `•` (bullet) только в списках
- Hairline-линии `─` — нет, использовать border CSS

## Шрифты и типографика

- Основной — Inter (`var(--font-inter)`)
- Заголовки H1-H2 — `font-weight: 800`, `letter-spacing: -0.03em`, `line-height: 1.15-1.2`
- Body — `font-weight: 400`, `line-height: 1.5-1.7`, `color: #6b7280`
- Подписи small — `font-size: 11-12px`, `color: #9ca3af`
- НИКАКИХ Comic Sans, никаких stock-шрифтов

## Промт-инжиниринг для image-моделей

Когда в промте Banana / Gemini Image передаются маркеры — **НЕ использовать английские слова которые могут быть срисованы**:
- ❌ `TITLE:`, `USP:`, `HEADING:`, `Caption:`
- ✅ Передавать тексты ТОЛЬКО как кириллица в `«»`-кавычках

Banana буквально интерпретирует английские слова в промте и может нарисовать их на финальной картинке. Это уже было в проде — слово `USP:` оказалось на карточке Certina.

## Архитектура (актуальная на 2026-05)

- API: FastAPI + SQLAlchemy async, Postgres на сервере (host.docker.internal:5432)
- Web: Next.js 15 App Router
- AI: Vertex AI (Gemini 3 Pro Image) через `google-genai` SDK, location=global
- Сервер: США (Atlanta, Baxet) — нужен из-за блокировки Vertex AI для российских IP
- Деплой: Docker Compose на одном хосте (`/var/www/aiviso/`)

При обсуждении переезда — помнить: **Vertex AI не работает с РФ-IP**, поэтому переезд в РФ невозможен без отказа от Gemini.

## Размеры карточек маркетплейсов (генерация и пресеты скачивания)

**Wildberries и Ozon оба используют аспект 3:4 для карточек товара.** Не путать с устаревшими рекомендациями про квадрат 1:1 для Ozon — на практике селлеры везде заливают 3:4 (вертикальный формат), это даёт максимальную площадь в листинге.

Целевые размеры пресетов скачивания:
- **Wildberries:** 900 × 1200
- **Ozon:** 900 × 1200 (тот же 3:4, **не** 1000 × 1000)

Если в коде где-то остался пресет `1000 × 1000` для Ozon — это баг, исправлять на `900 × 1200`.

Генерация в Gemini Image — `aspect_ratio: "3:4"` (1200 × 1600 на выходе), потом ресайз/кроп под пресеты выгрузки.
